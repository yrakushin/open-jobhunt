# Стиль писем (stop-slop + личные правила)

> Пример стилевого гайда для генерации сопроводительных писем.
> Скопируйте в `skills/stop-slop-writing.md` (`cp skills/stop-slop-writing.example.md skills/stop-slop-writing.md`)
> и адаптируйте под свой голос. Реальный файл в `.gitignore`.
>
> Основано на [hardikpandya/stop-slop](https://github.com/hardikpandya/stop-slop).

## Обязательные правила

- тело письма начинай **с маленькой буквы** (неформальный тон)
- **имена, фамилии, названия компаний** пиши с большой буквы
- **избегай двоеточий**
- **минимум тире** (ни длинных, ни средних)
- общайся по-человечески, без корпоративного «ИИ-слопа»

## Запрещённые паттерны

- зачины «рад представить», «хочу отметить», «в современном мире», «уникальная возможность»
- наречия на -о (реально, действительно, фундаментально)
- пассивный залог без субъекта
- «не X, а Y» — пиши сразу Y
- риторические вопросы в конце
- списки из трёх пунктов подряд
- фразы «стоит отметить», «важно понимать», «в заключение»
- цитатные «мудрые» финалы абзацев
- meta-комментарии про письмо

## Как писать

- активный залог, ты = кандидат делает что-то конкретное
- 70–90 слов, 2–4 варианта шаблона под тип роли
- факты только из резюме, цифры где есть
- под роль: акцент на use cases и delivery (укажите свои ключевые темы)
- подпись берётся из `config.yaml` → `profile.signature`

## Самопроверка перед отправкой

- есть ли двоеточия? убери
- есть ли тире? замени на точку или запятую
- звучит как ChatGPT? перепиши проще
- первая буква тела письма маленькая? да
- имя в подписи с большой? да
