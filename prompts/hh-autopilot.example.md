# HH.ru автопоиск — пример промпта для browser automation

> **Пример.** Это шаблон промпта для AI-агента, который автоматизирует отклики на hh.ru
> через браузер (Cursor Browser Tab / Claude computer-use и т.п.).
>
> Open JobHunt (`jobhunt run` / `jobhunt ui`) выполняет тот же цикл **без внешнего агента** —
> через Playwright + локальный LLM. Этот промпт нужен только если вы хотите повторить
> логику руками в другом агенте. Скопируйте в `prompts/hh-autopilot.md` и адаптируйте.
>
> В промпте нет персональных данных — все параметры берутся из `config.yaml` и
> `context/resume-profile.md` пользователя.

Ты автоматически ищешь работу на hh.ru через браузер и откликаешься на релевантные вакансии.
Действуй автономно, без уточняющих вопросов.

**Расписание и лимиты:** задаются в `config.yaml` (`search.max_applications_per_run`).
По умолчанию не больше 20 откликов за запуск. Достиг лимита — остановись, сохрани отчёт.

Перед стартом прочитай:
- `@context/resume-profile.md` — профиль, опыт, критерий соответствия
- `@skills/stop-slop-writing.md` — стиль сопроводительных писем
- `@config.yaml` — параметры поиска, фильтры, пороги

---

## ПОДГОТОВКА

1. Открой `https://hh.ru/applicant/resumes` в браузере.
2. Если не авторизован — **ничего не делай**, сообщи что нужен ручной логин и остановись.
3. Резюме для отклика: название из `config.yaml` → `profile.resume_title`.
   Возьми `resumeHash` из URL `/resume/<hash>` по названию резюме.
4. Рабочая папка отчётов: `reports/` в корне репозитория.

---

## ПАРАМЕТРЫ ПОИСКА (из config.yaml)

- Регион: `search.area` (1 = Москва)
- Период: `search.period_days` (0 = без ограничения по дате)
- Порог зарплаты: `search.min_salary_rub`. Если в вакансии указана зарплата и верхняя
  планка ниже порога — не откликаться. Если зарплата не указана — можно рассматривать.
- Лимит откликов за запуск: `search.max_applications_per_run` (или флаг `--limit`).
- Лимит откликов на одну компанию: `search.max_per_company` (0 = без лимита).
- Чёрный список: `filters.blacklist_employers`. Белый список: `filters.whitelist_employers`.
- Пожелания: `search.vacancy_preferences` (формат, локация, доп. роли).

## ФИЛЬТР (match ≥ llm_min_percent)

Используй критерии из `@context/resume-profile.md` и методологию `@context/match-methodology.md`.

**Режим «По роли» (`search.role_filter_mode: role_lock`):**
- отклик только на роли из `profile.resume_title` + доп. роли из `vacancy_preferences`
- если название не подходит, но обязанности пересекаются с профилем
  ≥ `search.role_duties_min_percent` (по умолчанию 40%) — отклик

**Режим «Свободно» (`search.role_filter_mode: off`):**
- без ограничения по названию роли
- отклик при пересечении опыта/обязанностей ≥ `search.free_experience_min_percent` (30%)

**СТОП-РОЛИ (никогда):** стажировки, internship, trainee, junior, джун, «без опыта»,
DevRel, Product Marketing, чистый ML/Data/Backend IC, Sales/Marketing/HR.

**СТОП-РАБОТОДАТЕЛИ (никогда):** `filters.blacklist_employers`, МФО/микрофинансы,
ИП/ФИО в названии компании.

Для каждой вакансии после чтения описания оцени match 0–100%. Отклик только при
≥ `search.llm_min_percent` (по умолчанию 30%). В отчёте указывай оценку match.

## НЕ ДУБЛИРОВАТЬ

1. Собери id откликов с `https://hh.ru/applicant/negotiations` → `vacancyId`.
2. Пропускай уже откликнутые.
3. На форме отклика признак дубля — «Вы откликнулись».

---

## ОТКЛИК

### Жёсткие правила

1. **Один vacancyId = один отклик.** Между вакансиями — пауза 1–2 с.
2. **Только прямой URL:** `https://hh.ru/applicant/vacancy_response?vacancyId=<id>&resumeHash=<hash>`.
   Этот URL сразу отправляет отклик. Сразу после перехода нажми
   «Сопроводительное письмо → Добавить» и заполни письмо — иначе отклик уйдёт без текста.
3. **Никогда** не жми «Откликнуться» на странице поиска, в карточках, в блоках «похожие вакансии».
4. **Без письма не отправлять.** Если поле письма недоступно — пропуск.
5. **После отправки проверь** в `negotiations`, что появился только один новый отклик.

### СТОП-УСЛОВИЯ (вынести в «Требует действия», не откликаться)

- тест/опросник/анкета на сайте или внешняя ссылка на тест
- капча / «подтвердите что вы не робот»
- требование сменить видимость резюме или настройки профиля
- обязательное прикрепление файла, которого нет
- нестандартная форма отклика (не удаётся отправить за 2 попытки)

При **капче или блокировке hh** на любом этапе — **останови весь запуск**, сохрани
промежуточный отчёт, сообщи что нужно вмешательство.

---

## ПИСЬМО

- 70–90 слов, вариант под тип роли
- факты из резюме с цифрами
- стиль из `@skills/stop-slop-writing.md` (маленькая буква, без двоеточий, минимум тире)
- если работодатель спрашивает зарплатные ожидания — ответь из `profile.salary_expectation`

## ОТЧЁТ

Собери Excel: `reports/Otclicki_hh_User_<YYYY-MM-DD>_<HH-mm>.xlsx`

### Лист «Отклики»
| статус | должность | компания | регион | зарплата | match% | ссылка | текст письма | время |

### Лист «Требует действия»
| причина | должность | компания | зарплата | match% | ссылка | что сделать вручную | заметки |

Причины: `тест на сайте`, `опросник`, `капча`, `смена видимости резюме`, `ошибка формы`,
`антибот при чтении`, `другое`.

### Лист «Пропущено»
| причина | должность | компания | зарплата | match% | ссылка |

Причины: `match <N%`, `зарплата ниже порога`, `стажёр/junior`, `дубль`,
`исключённый работодатель`, `МФО/микрофинансы`, `ИП/ФИО работодатель`,
`лимит компании`, `лимит откликов`, `стоп-роль`.

## ИТОГ В ЧАТЕ

Кратко без вопросов:
- сколько откликов отправлено
- сколько в «Требует действия» (с разбивкой по причинам)
- сколько пропущено (топ-3 причины)
- путь к Excel-файлу
- что требует решения вручную (логин, капча, тесты)
