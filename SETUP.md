# Настройка с нуля (если что-то сломалось)

> Краткий старт — в [`README.md`](README.md). Полная инструкция — в [`GUIDE.md`](GUIDE.md).
> Этот файл — для случаев, когда что-то пошло не так.

## Диагностика

| Что проверить | Ожидание |
|---------------|----------|
| Папка проекта | корень репозитория (куда вы его склонировали) |
| Python | 3.11+ (`python3 --version`) |
| Зависимости | `pip install -e .` выполнено в корне |
| Конфиг | `config.yaml` существует (`jobhunt init`) |
| Браузер | отдельный профиль Chromium (`~/.jobhunt/browser-profile`), не ваш Chrome |
| Окно | `jobhunt ui` — одно окно с вкладками «Панель» и «hh.ru» |

Универсальная проверка:

```bash
jobhunt doctor
```

Покажет: config, профиль, playwright, ollama, модель, профиль браузера.

## Шаг 1. Установить зависимости

**macOS / Linux:**

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
jobhunt setup
```

**Windows (PowerShell):**

```powershell
.\scripts\install.ps1
.\.venv\Scripts\Activate.ps1
```

Или вручную: `python -m venv .venv`, `pip install -e .`, `jobhunt setup`.

---

## Шаг 2. Подготовить персональные файлы

В репозитории — только примеры `*.example.md`. Реальные файлы в `.gitignore`:

```bash
jobhunt init      # config.yaml, пустой context/cover-letter.md и остальное из *.example
# или вручную:
cp context/resume-profile.example.md context/resume-profile.md
cp skills/stop-slop-writing.example.md skills/stop-slop-writing.md
```

Откройте `context/resume-profile.md` и заполните профиль (роль, опыт, навыки, критерий
соответствия, точное название резюме на hh.ru). См. `GUIDE.md` §8.

---

## Шаг 3. Залогиниться на hh.ru

Запустите панель:

```bash
jobhunt ui
```

В открывшемся окне:
1. Нажмите **«Войти на hh.ru»** (или вкладку **hh.ru**).
2. Войдите вручную (телефон/почта, 2FA, капча — сами).
3. Убедитесь, что видите своё резюме.
4. Карточка **«Сессия hh.ru»** должна показать «Активна».

Сессия хранится в `~/.jobhunt/browser-profile` и переживает перезапуск `jobhunt ui`.
Это **отдельный профиль**, не ваш системный Chrome — логин в Chrome боту не поможет.

Альтернатива без панели: `jobhunt login` (открывает окно для входа один раз).

---

## Шаг 4. Загрузить резюме и задать поиск

В панели:
1. Карточка **«Резюме»** → **«Заменить»** → выберите `.md`/`.txt`/`.pdf` → **«Загрузить»**.
2. На hh.ru откройте своё резюме → **«Подходящие вакансии»** → скопируйте URL из адресной строки.
3. Вставьте URL в карточку **«Поиск на hh.ru»** → **«Сохранить ссылку»**.

Если ссылку не задать — бот ищет по `resume_title` (авто-поиск) и keyword-запросам.

---

## Шаг 5. Выбрать режим и проверить

1. Карточка **«Соответствие вакансии»** — **«По роли»** или **«Свободно»** (см. `GUIDE.md` §4).
2. Заполните **«Зарплата и пожелания»** и **«Фильтр компаний»**.
3. Нажмите **«Тест»** (dry-run) — бот пройдёт цикл без отправки откликов.
4. Проверьте отчёт в карточке **«Отчёты»** (или `reports/Otclicki_hh_User_*.xlsx`).

Если всё устраивает — **«Запустить отклики»** (двойной клик для подтверждения).

## Частые проблемы

**«Не авторизован на hh.ru»**
→ Войдите в окне `jobhunt ui` (вкладка hh.ru) или `jobhunt login`. Сессия в `~/.jobhunt/browser-profile`,
не в Chrome. См. `GUIDE.md` §12.

**Капча останавливает прогон**
→ Это задумано. Пройдите капчу вручную во вкладке hh.ru, перезапустите прогон. Бот не обходит
антибот. См. `GUIDE.md` §12.

**Ollama не отвечает / `ollama down`**
→ `ollama serve` в отдельном терминале, `ollama pull qwen2.5:7b`, проверьте `jobhunt doctor`.
См. `GUIDE.md` §12.

**0 откликов / 0 match**
→ Проверьте `resume_title`, `resume_search_url`, `llm_min_percent`, режим «По роли»/«Свободно»,
заполненность профиля. Запустите dry-run, смотрите лист «Пропущено». См. `GUIDE.md` §12.

**Порт 8787 занят**
→ `jobhunt ui --port 8788` или закройте старый экземпляр. См. `GUIDE.md` §12.

**Профиль/письмо не загружаются**
→ Форматы `.md`/`.txt`/`.pdf`. PDF с <80 символами отклоняется. Лимиты: резюме 5 МБ, письмо 500 КБ.

**VPN мешает hh.ru**
→ Отключите VPN или нажмите «Я не использую VPN» во вкладке hh.ru в окне панели.

Подробный troubleshooting — `GUIDE.md` §12.
