## 1. Что это и зачем

**Open JobHunt** — локальный CLI/UI-ассистент, который автоматизирует рутину поиска работы на hh.ru:
поиск релевантных вакансий, оценка соответствия (match), генерация сопроводительных писем и отправка откликов.

**Зачем:** компании автоматизируют ATS и массовые отказы, кандидат тратит часы на рутину.
Open JobHunt ускоряет отбор и отклик, оставляя решение (на какую вакансию откликаться) за вами.

**Что делает:**
- ищет вакансии на hh.ru (подбор по резюме или keyword-запросы);
- фильтрует: чёрный/белый список компаний, стоп-роли, МФО, ИП, зарплата, дубли откликов;
- оценивает соответствие (rule-based + LLM через Ollama/OpenAI);
- генерирует сопроводительное письмо по вашему профилю и стилю;
- отправляет отклик через Playwright (сохранённая сессия hh.ru);
- пишет Excel-отчёт: отклики / требует действия / пропущено.

**Что НЕ делает (важно):**
- **не облако и не SaaS** — всё работает на вашей машине, сервер слушает только `127.0.0.1`;
- **не обходит капчу и антибот** — при капче останавливается и просит вмешательство;
- **не заполняет тесты/опросники** — выносит такие вакансии в «Требует действия»;
- **не откликается массово без письма** — каждый отклик идёт с осмысленным текстом;
- **не нарушает правила hh.ru** — паузы между откликами, лимиты за запуск;
- **v0.2: только hh.ru** (другие площадки — в планах, через `jobhunt/adapters/`).

## 2. Требования

| Что | Версия / Notes |
|-----|----------------|
| ОС | macOS, Linux, Windows |
| Python | 3.11+ |
| Playwright | ставится через `jobhunt setup` (`playwright install chromium`) |
| Ollama (рекомендуется) | локальный LLM-сервер, https://ollama.com |
| Браузер | Chromium (ставится Playwright) — отдельный профиль, не ваш Chrome |
| Учётная запись hh.ru | обычный логин, который вы проходите вручную один раз |

**Опционально:**
- внешний OpenAI-совместимый API (если не хотите локальный Ollama) — см. [§10](#10-llm-и-приватность).

## 3. Установка с нуля

```bash
git clone https://github.com/yrakushin/open-jobhunt.git
cd open-jobhunt

python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

pip install -e .

jobhunt init      # создаёт config.yaml из config.example.yaml
jobhunt setup     # ставит Chromium + тянет модель Ollama (qwen2.5:7b)
jobhunt ui        # открывает графическую панель (рекомендуется)
```

**Windows (PowerShell, один скрипт):**

```powershell
git clone https://github.com/yrakushin/open-jobhunt.git
cd open-jobhunt
.\scripts\install.ps1
.\.venv\Scripts\Activate.ps1
jobhunt login
jobhunt ui
```

Если встроенное окно Chromium не стартует: `jobhunt ui --browser` (панель в Edge/Chrome).

> Замените `YOUR_USERNAME` на свой GitHub-форк. Запускать можно и без venv,
> но venv изолирует зависимости.

**Что делает `jobhunt init`:** копирует `config.example.yaml` → `config.yaml`. Другие файлы
не создаёт — примеры (`*.example.md`) уже в репо, их нужно скопировать вручную (см. [§4](#4-первый-запуск-пошагово)).

**Что делает `jobhunt setup`:** `python -m playwright install chromium` + `ollama pull <model>`
(модель из `config.yaml` → `llm.ollama.model`, по умолчанию `qwen2.5:7b`). Если Ollama не установлен —
печатает ссылку на https://ollama.com и продолжит (LLM будет недоступен, но rule-based оценка работает).

## 4. Первый запуск пошагово

### Шаг 1. Подготовьте персональные файлы

В репозитории лежат только примеры (`*.example.md`). Реальные файлы в `.gitignore` — их не нужно коммитить.

```bash
cp context/resume-profile.example.md context/resume-profile.md
cp context/cover-letter.example.md   context/cover-letter.md
cp skills/stop-slop-writing.example.md skills/stop-slop-writing.md
```

Откройте `context/resume-profile.md` и заполните свой профиль (см. [§8](#8-профиль-кандидата)).
Откройте `skills/stop-slop-writing.md` и адаптируйте стиль (см. [§9](#9-стиль-писем)).
Письмо `context/cover-letter.md` можно оставить шаблоном или очистить — тогда письмо сгенерирует LLM.

### Шаг 2. Запустите панель

```bash
jobhunt ui
```

Откроется одно окно Chromium с двумя вкладками: **Панель** (управление) и **hh.ru** (реальный сайт).
Панель также доступна по адресу `http://127.0.0.1:8787`.

### Шаг 3. Войдите на hh.ru

- В панели нажмите **«Войти на hh.ru»** (или вкладку **hh.ru**).
- В открывшейся вкладке hh.ru войдите как обычно (телефон/почта, 2FA, капча — вручную).
- Сессия сохранится в постоянном профиле Chromium (`~/.jobhunt/browser-profile`) и переживёт
  перезапуск `jobhunt ui`.
- Карточка **«Сессия hh.ru»** в панели покажет «Активна».

Альтернатива без панели — `jobhunt login` (открывает окно для входа один раз).

### Шаг 4. Загрузите резюме

- Карточка **«Резюме»** → **«Заменить»** → выберите `.md` / `.txt` / `.pdf` → **«Загрузить»**.
- Файл читается локально, текст сохраняется в `context/resume-profile.md` (ваш `profile_path`).
- Бот автоматически угадает `resume_title` из первой строки/заголовка и запишет в `config.yaml`.
- Карточка **«Резюме»** в статусе покажет название.

> Файл никуда не уходит — только на диск и (позже) в локальный Ollama для оценки/письма.

### Шаг 5. Настройте ссылку поиска (рекомендуется)

На hh.ru откройте своё резюме → кнопка **«Подходящие вакансии»** → скопируйте URL из адресной строки
(вида `https://hh.ru/search/vacancy?resume=…`) → вставьте в карточку **«Поиск на hh.ru»** →
**«Сохранить ссылку»**. Это самый точный источник вакансий.

Если ссылку не задать — бот ищет по `resume_title` (авто-поиск) и keyword-запросам из `search.queries`.

### Шаг 6. Выберите режим соответствия

Карточка **«Соответствие вакансии»** — два режима:

- **«По роли» (`role_lock`)** — отклик только на роли из `resume_title` + доп. роли из «Пожеланий».
  Если название не подходит, но обязанности пересекаются с профилем ≥ `role_duties_min_percent`
  (по умолчанию 40%) — отклик. Жёстко отсекает архитекторов, разработчиков, sales и т.п.
- **«Свободно (`off`)** — без ограничения по названию роли; отклик при пересечении
  опыта/обязанностей ≥ `free_experience_min_percent` (по умолчанию 30%).

> **Важно:** итоговый порог отбора (число % match, выше которого отправляется отклик) —
> `search.llm_min_percent` в `config.yaml` (по умолчанию 30). В UI его нет, меняется в файле.

### Шаг 7. Заполните пожелания, зарплату, фильтры

- **«Зарплата и пожелания»**: минимальная зарплата (0 = не фильтровать) + свободные пожелания
  (удалёнка, доп. роли и т.д.).
- **«Фильтр компаний»**: «Не откликаться» (чёрный список, по строке на компанию) и
  «Только эти компании» (белый список; пусто = без ограничения).

### Шаг 8. Тестовый прогон (dry-run)

- В карточке **«Запуск»** укажите число откликов (например 5).
- Нажмите **«Тест»** — бот пройдёт весь цикл (поиск, оценка, письма, отчёт), **но не отправит отклики**.
- В карточке **«Ход прогона»** смотрите лог. В карточке **«Отчёты»** появится `.xlsx`.

### Шаг 9. Реальный запуск

- Нажмите **«Запустить отклики»** — кнопка потребует **двойного клика** для подтверждения
  (первый клик — «Точно отправить N?», второй в течение 4 с — старт).
- Между откликами бот делает паузу 8–35 с, чтобы не нарушать лимиты hh.ru.
- Достиг лимита или столкнулся с капчей — остановится сам. Можно остановить вручную: **«Стоп»**.

### Шаг 10. Отчёт

В карточке **«Отчёты»** — список `.xlsx`. **«Скачать»** — скачать файл, **«Открыть»** — открыть
в системном приложении (Excel/Numbers/LibreOffice). См. [§11](#11-отчёты-xlsx).

## 5. Команды CLI

Все команды вызываются как `jobhunt <команда> [флаги]`. Точка входа — `pyproject.toml` →
`[project.scripts] jobhunt = "jobhunt.cli:main"`.

### `jobhunt init`
Создаёт `config.yaml` из `config.example.yaml`. Если `config.yaml` уже есть — не перезаписывает.

### `jobhunt setup`
- `python -m playwright install chromium` — ставит браузер для автоматизации.
- `ollama pull <model>` — тянет модель из `config.yaml` → `llm.ollama.model` (по умолчанию `qwen2.5:7b`),
  если установлен `ollama`. Если нет — печатает ссылку на https://ollama.com.

### `jobhunt login`
Открывает Chromium на `https://hh.ru/account/login`. Вы входите вручную, закрываете окно / Ctrl+C —
сессия сохраняется в `~/.jobhunt/browser-profile`. Нужен один раз (потом сессия переиспользуется).

### `jobhunt run [--dry-run] [--limit N] [--config PATH]`
Полный цикл: поиск → фильтр → оценка → письмо → отклик → отчёт.

| Флаг | Назначение |
|------|------------|
| `--dry-run` | Без отправки откликов — только отбор и письма (отчёт всё равно пишется) |
| `--limit N` | Сколько откликов за запуск (переопределяет `search.max_applications_per_run`) |
| `--config PATH` | Альтернативный config-файл |

После прогона печатает таблицу «Итог запуска»: отправлено / требует действия / пропущено / путь к отчёту.
При ошибках — exit code 1.

> `jobhunt run` использует отдельный профиль `~/.jobhunt/browser-profile-auto`, чтобы фоновая
> автоматизация не конфликтовала с интерактивным окном `jobhunt ui`. В UI прогон выполняется
> in-process (в том же окне), либо как subprocess — автоматически.

### `jobhunt doctor`
Проверка окружения. Строит таблицу «Проверка / Статус»:
- `config.yaml` — есть или используется `config.example.yaml`;
- `profile` — файл `profile.profile_path` существует;
- `playwright` — импортируется;
- `ollama` — отвечает на `http://127.0.0.1:11434/api/tags`;
- `model <name>` — модель установлена в Ollama;
- `browser profile` — каталог `browser.profile_dir` существует.

> `doctor` не проверяет API-ключ OpenAI, ссылку поиска и папку отчётов.

### `jobhunt ui [--host HOST] [--port PORT] [--browser]`
Графическая панель. По умолчанию открывает окно Chromium с двумя вкладками (Панель + hh.ru).
- `--host` (по умолчанию `127.0.0.1`) — только loopback, не exposing в сеть;
- `--port` (по умолчанию `8787`);
- `--browser` — открыть панель в системном браузере (без окна Chromium; hh.ru-вкладку тогда
  нужно открывать отдельно).

Если порт занят и отвечает — откроет существующий экземпляр. Если занят и не отвечает — перезапустит.

### `jobhunt --version`, `jobhunt --help`
Версия пакета / список команд.

## 6. Веб-панель (jobhunt ui) — каждый блок

Панель — одностраничный дашборд на FastAPI, отдаётся на `http://127.0.0.1:8787`. Разметка —
`jobhunt/web/static/index.html`, логика — `app.js`. Сверху: бренд, переключатель вкладок
**Панель**/**hh.ru**, кнопки **↻ Обновить** и **📖 Инструкция**. Ниже — две статус-карточки
и сетка карточек в две колонки.

### Статус-ряд (две карточки)
- **Сессия hh.ru** — «Активна» / «Откройте hh.ru». Зелёная, если есть профиль браузера или
  файл состояния `~/.jobhunt/browser-state.json`.
- **Резюме** — название резюме или «не загружено». Зелёная, когда профиль загружен и есть `resume_title`.

### Левой колонка

**Резюме** — выбор файла (`.md`/`.txt`/`.pdf`), загрузка в `profile.profile_path`, авто-угадывание
`resume_title`. Лимит 5 МБ. Текст хранится локально. Кнопки: **Заменить**, **Загрузить**.

**Сопроводительное письмо** — текстовое поле с шаблоном. Кнопки: **Сохранить письмо**
(пишет в `context/cover-letter.md`), **Из файла .md/.txt** (загружает текст в поле, потом
нужно нажать «Сохранить письмо»). Поддерживает плейсхолдеры `{title}`, `{company}`, `{signature}`,
`{должность}`, `{компания}`. Если шаблон непустой — LLM не вызывается для письма.

**Зарплата и пожелания** — `Мин. зарплата, ₽` (0 = не фильтровать) + `Свои пожелания к вакансиям`
(свободный текст). Кнопка **Сохранить**. Пожелания уходят в LLM при оценке и парсятся на доп. роли
в режиме «По роли» (фразы «откликайся ещё на …», «роли: …»).

**Фильтр компаний** — `Не откликаться` (чёрный список, по строке на компанию) и `Только эти компании`
(белый список). Кнопка **Сохранить фильтры**. Совпадение — подстрока после нормализации
(ООО/ПАО убираются, без учёта регистра).

### Правая колонка

**Поиск на hh.ru** — поле для ссылки `https://hh.ru/search/vacancy?resume=…`. Кнопки:
**Сохранить ссылку**, **Открыть на hh.ru**. Пустая ссылка = авто-поиск по `resume_title`.
Валидируется префикс `https://hh.ru/search/vacancy`.

**Соответствие вакансии** — чипы **Свободно** (`off`) / **По роли** (`role_lock`). Под ними —
инфо-строка с ролями из резюме и порогом обязанностей. Это единственный UI-контроль соответствия;
`llm_min_percent` меняется только в `config.yaml`. См. [§7](#7-configyaml--все-поля), [§10](#10-llm-и-приватность).

**Запуск** — поле `Откликов за запуск` (1–500) и кнопки:
- **Войти на hh.ru** — открывает вкладку hh.ru;
- **Тест** — dry-run (`/api/run?...&dry_run=true`);
- **Запустить отклики** — реальный прогон, двойной клик для подтверждения;
- **Все по зарплате** — реальные отклики без match и фильтра роли; учитываются мин. зарплата,
  чёрный список и лимиты (`--salary-only` / `salary_only=true`), двойной клик;
- **Стоп** — мягкая остановка после текущей вакансии.

Перед стартом (Тест / Запустить / Все по зарплате) панель тихо сохраняет фильтры, письмо, ссылку, пожелания и режим.

**Ход прогона** — бейдж статуса (Ожидание / Старт / Работает / Готово / Ошибка), степпер из 4 шагов
(Поиск → Оценка → Отклик → Готово) и тёмный лог-блок с временными метками. Зелёные строки —
отправлено/dry-run/итог, красные — ошибки.

**Отчёты** — список последних 10 `.xlsx`. Каждая строка: имя, дата, **Скачать**, **Открыть**.
Пусто → «Пока нет отчётов». Path-traversal в API отчётов закрыт (путь проверяется внутри `reports_dir`).

### Модальная инструкция
Кнопка **📖 Инструкция** открывает пошаговый тур по 9 карточкам (стрелки Назад/Далее, Esc — закрыть).

## 7. config.yaml — все поля

`config.yaml` создаётся из `config.example.yaml`. Относительные пути разрешаются от корня проекта.
В UI сервер пишет в `config.yaml` (или в `config.example.yaml`, если `config.yaml` нет).

### `profile`
| Поле | Тип | По умолчанию | Назначение |
|------|-----|--------------|-----------|
| `resume_title` | string | `"Название резюме на hh.ru"` | Точное название резюме на hh.ru; для авто-поиска и ролей в «По роли» |
| `profile_path` | path | `context/resume-profile.md` | Ваш профиль (.md/.txt), копируется из `.example.md` |
| `letter_template_path` | path | `context/cover-letter.md` | Шаблон письма; пустой файл → письмо генерирует LLM |
| `letter_style_path` | path | `skills/stop-slop-writing.md` | Стиль писем для LLM, копируется из `.example.md` |
| `salary_expectation` | string | `""` | Что отвечать на вопрос о зарплате (в LLM-промпт) |
| `signature` | string | `""` | Подпись в письме (имя / контакты на ваше усмотрение) |

### `search`
| Поле | Тип | По умолчанию | Назначение |
|------|-----|--------------|-----------|
| `site` | string | `hh_ru` | Площадка (только `hh_ru` в v0.2) |
| `area` | int | `1` (Москва) | Код региона hh.ru (`&area=`) |
| `items_per_page` | int | `100` | Вакансий на страницу поиска |
| `queries` | list[str] | `["Product Manager","Product Owner"]` | Keyword-запросы; только когда `resume_search_url` пуст |
| `use_resume_search` | bool | `true` | Использовать подбор по резюме |
| `resume_search_url` | url | `""` | Ссылка «Подходящие вакансии»; пусто = авто-поиск по `resume_title` |

## 8. Профиль кандидата

Файл `context/resume-profile.md` — ваш профиль в markdown. Скопируйте из `context/resume-profile.example.md`
и заполните. Можно загрузить через UI (карточка «Резюме», `.md`/`.txt`/`.pdf`).

**Что важно заполнить:**
- целевая роль и специализации (для режима «По роли» и авто-поиска);
- опыт с цифрами («было → стало») — факты идут в письма;
- навыки / домены / технологии — LLM оценивает по ним соответствие;
- **критерий соответствия** — блок must/preferred/blocker (см. `context/match-methodology.md`);
- **название резюме на hh.ru** — бот ищет его при авто-поиске и выводит из него роли в «По роли».

**Что НЕ нужно писать в профиль:** контакты (телефон, email, telegram) — Open JobHunt не
использует их при отклике. Подпись и контакты для писем задаются в `config.yaml` → `profile.signature`
и в шаблоне `context/cover-letter.md`.

> Профиль в `.gitignore` — не коммитится. Загрузка через UI пишет текст в `profile.profile_path`.

## 9. Стиль писем

Файл `skills/stop-slop-writing.md` — стилевой гайд для генерации писем (когда `context/cover-letter.md`
пустой, и письмо пишет LLM). Скопируйте из `skills/stop-slop-writing.example.md` и адаптируйте.

**Базовые правила (из примера):**
- тело письма с маленькой буквы (неформальный тон);

- без двоеточий, минимум тире;

Если задан шаблон `context/cover-letter.md` (непустой) — письмо формируется подстановкой
плейсхолдеров `{title}`/`{company}`/`{signature}` **без LLM**.

## 10. LLM и приватность

Open JobHunt использует LLM для двух задач: оценка соответствия (match) и генерация письма
(если нет шаблона). Поставщик — `config.yaml` → `llm.provider`.

### Ollama (по умолчанию, локально)
- `provider: ollama`, модель `qwen2.5:7b`, адрес `http://127.0.0.1:11434`.
- Профиль резюме, описания вакансий и пожелания уходят **только на вашу машину**.
- Бесплатно, оффлайн. Рекомендуемый вариант.

### OpenAI-совместимый API (опционально)
- `provider: openai` или `fallback_to_api: true` (фоллбэк при недоступном Ollama).
- Ключ — через env-переменную (имя из `llm.openai.api_key_env`, по умолчанию `JOBHUNT_API_KEY`):

  ```bash
  export JOBHUNT_API_KEY=sk-...
  ```

- **Внимание:** при `provider: openai` или `fallback_to_api: true` профиль резюме, описания
  вакансий и пожелания уходят на `llm.openai.base_url` (по умолчанию `https://api.openai.com/v1`).
  Это ожидаемая опция, но данные покидают вашу машину. Используйте осознанно.

### `provider: none`
- LLM не используется. Оценка только rule-based (`rule_match_score`), письмо — из шаблона
  или жёстко зашитый фоллбэк. Подходит для проверки фильтров без LLM.

### Что куда уходит — кратко
| Конфиг | hh.ru | Локальный Ollama | Внешний API |
|--------|-------|------------------|-------------|
| `provider: ollama`, `fallback_to_api: false` (по умолчанию) | поиск + отклики (неизбежно) | профиль, вакансии, письма | — |
| `provider: openai` или `fallback_to_api: true` | поиск + отклики | — | профиль, вакансии, письма |
| `provider: none` | поиск + отклики | — | — |

> Отклик на hh.ru (форма с письмом) — неизбежная отправка на hh.ru, это и есть суть отклика.

## 11. Отчёты xlsx

После каждого прогона (включая `--dry-run`) в `paths.reports_dir` (по умолчанию `reports/`)
пишется `Otclicki_hh_User_<YYYY-MM-DD>_<HH-MM>.xlsx`. В UI — карточка «Отчёты»: **Скачать** / **Открыть**.

**Листы:**

- **«Отклики»** — `статус | должность | компания | регион | зарплата | match% | ссылка | текст письма | время`.
  Статус = «отклик отправлен» (или `dry_run` при тестовом прогоне).
- **«Требует действия»** — `причина | должность | компания | зарплата | match% | ссылка | что сделать вручную | заметки`.
  Причины: тест на сайте, опросник, капча, смена видимости резюме, ошибка формы, антибот, другое.
- **«Пропущено»** — `причина | должность | компания | зарплата | match% | ссылка`.
  Причины: match <N%, зарплата ниже порога, стажёр/junior, дубль, исключённый работодатель,
  МФО/микрофинансы, ИП/ФИО работодатель, лимит компании, лимит откликов, стоп-роль.

**Как открыть:** кнопка «Открыть» в UI (вызывает системное приложение) или просто двойной клик
по файлу. В отчёте содержится текст письма — открывайте локально, не публикуйте.

> Отчёты в `.gitignore` (`reports/*.xlsx`). В репозитории только `reports/.gitkeep`.

## 12. Troubleshooting

**«Не авторизован на hh.ru»**
→ Войдите вручную в окне `jobhunt ui` (вкладка hh.ru) или `jobhunt login`. Сессия в отдельном
профиле Chromium (`~/.jobhunt/browser-profile`), не в вашем Chrome. После входа карточка
«Сессия hh.ru» станет «Активна».

**Капча / «Капча не пройдена»**
→ Бот останавливает прогон и пишет об этом. Откройте вкладку hh.ru в окне `jobhunt ui`,
пройдите капчу вручную, перезапустите прогон. Open JobHunt не обходит антибот.

**Ollama не запущен / `ollama down`**
→ Запустите `ollama serve` в отдельном терминале. Проверьте `jobhunt doctor` (строка `ollama`).
Модель должна быть скачана: `ollama pull qwen2.5:7b` (или вашу из `config.yaml`).
Без Ollama работает rule-based оценка (`provider: none` или LLM недоступен — причина «LLM недоступен»).

**0 откликов / «0 match»**
→ Проверьте: `resume_title` совпадает с названием на hh.ru; `resume_search_url` валиден;
`llm_min_percent` не завышен (по умолчанию 30); режим «По роли» не слишком строг
(попробуйте «Свободно»); профиль заполнен (карточка «Резюме» зелёная). Запустите `--dry-run`
и смотрите лист «Пропущено» — там причины.

**Отчёт не открывается**
→ В UI кнопка «Открыть» вызывает `open` (macOS) / `xdg-open` (Linux) / `os.startfile` (Windows).
Если не работает — скачайте («Скачать») и откройте вручную. Файл лежит в `reports/`.

**Порт 8787 занят**
→ `jobhunt ui` сам проверит: если отвечает — откроет существующий экземпляр; если занят и не
отвечает — перезапустит (macOS/Linux: `lsof`, Windows: `netstat` + `taskkill`). Можно запустить
на другом порту: `jobhunt ui --port 8788`.

**Windows: окно Chromium не открывается**
→ Запустите `jobhunt ui --browser` — панель откроется в Edge/Chrome. Убедитесь, что выполнили
`playwright install chromium` (или `.\scripts\install.ps1`). Антивирус иногда блокирует Playwright —
добавьте папку проекта и `%USERPROFILE%\.jobhunt` в исключения.

**Windows: `UnicodeEncodeError` / `charmap` / символ `→` в логе**
→ Обновите до последней версии с GitHub (`git pull` или заново `git clone`). В v0.2.1+ лог из
панели не пишется в cp1251-консоль. Перезапустите `jobhunt ui` после обновления.

**Профиль/письмо не загружаются**
→ Форматы `.md`/`.txt`/`.pdf`. PDF с <80 символами текста отклоняется («Не удалось извлечь текст»).
Лимиты: резюме 5 МБ, письмо 500 КБ. Кодировка utf-8/cp1251/latin-1.

**Бот откликается на ненужные роли**
→ Включите режим «По роли» (карточка «Соответствие вакансии»). Добавьте стоп-роли/домены в
профиль (`context/resume-profile.md`, блок `blocker`). Пополните чёрный список компаний.
Поднимите `llm_min_percent` в `config.yaml`.

**Бот пропускает релевантные**
→ Попробуйте «Свободно». Снизьте `llm_min_percent` (но не ниже разумного). Проверьте, что
доп. роли есть в «Пожеланиях» (фраза «откликайся ещё на …»). Уберите слишком широкие записи
из чёрного списка (например короткие слова типа «aero» ловят лишнее).

**Сессия слетела после перезапуска**
→ Профиль `~/.jobhunt/browser-profile` сохраняется. Если удалили — войдите снова через
`jobhunt login` или вкладку hh.ru в `jobhunt ui`.

## 13. Безопасность

- **Данные локально.** Профиль, письма, фильтры, отчёты, сессия — на вашей машине.
  В `.gitignore`: `config.yaml`, `context/resume-profile.md`, `context/cover-letter.md`,
  `reports/*.xlsx`, `~/.jobhunt/` и др.
- **Сервер только на loopback.** `jobhunt ui` слушает `127.0.0.1:8787` — не доступен из сети.
  CORS отключён, авторизации нет (безопасно только на loopback). **Не меняйте host на `0.0.0.0`**
  и не включайте CORS без понимания — любой сайт в браузере сможет дёргать API.
- **Секреты через env.** API-ключ — в переменной окружения (`JOBHUNT_API_KEY` по умолчанию),
  не в `config.yaml` и не в git. Ключ не логируется, передаётся только в заголовке `Authorization`.
- **Телеметрии нет.** В коде нет analytics/Sentry/PostHog. Единственный внешний ресурс UI —
  Google Fonts (`fonts.googleapis.com`); если критично — можно убрать из `index.html`.
- **Path-traversal закрыт.** API отчётов проверяет, что путь остаётся внутри `reports_dir`.
- **Что уходит на hh.ru (неизбежно).** Поиск вакансий и форма отклика с письмом — это и есть
  работа бота. Внешние редиректы hh.ru бот не проходит автоматически (помечает «внешний сайт —
  заполните вручную»).
- **Что уходит в LLM.** См. [§10](#10-llm-и-приватность). По умолчанию (Ollama) — только на вашу машину.

## 14. FAQ для AI-агента

> Раздел для AI-агентов (Cursor, Claude Code, Copilot), отвечающих на вопросы пользователя.
> Краткие ответы на типовые вопросы. Подробности — в соответствующих разделах выше.

**«Как установить?»** → `git clone … && cd open-jobhunt && pip install -e . && jobhunt init && jobhunt setup && jobhunt ui`. См. §3.

**«Как запустить первый раз?»** → Скопируйте `*.example.md` в реальные файлы, `jobhunt ui`, войдите на hh.ru, загрузите резюме, задайте ссылку поиска, dry-run, затем запуск. См. §4.

**«Где мои данные / они уходят в облако?»** → Нет. Всё локально, сервер на `127.0.0.1`. В облако уходит только при `provider: openai`/`fallback_to_api: true`. См. §13, §10.

**«Зачем двойной клик на "Запустить отклики"?"**» → Подтверждение отправки реальных откликов. Первый клик — «Точно отправить N?», второй в течение 4 с — старт.

**«Чем отличается Тест от Запустить?"**» → Тест = `--dry-run`: весь цикл без отправки откликов, отчёт пишется. Запустить = реальные отклики. См. §5.

**«Что значит "По роли" и "Свободно"?"**» → «По роли» (`role_lock`): отклик только на роли из `resume_title` + доп. роли из пожеланий, иначе ≥40% обязанностей. «Свободно» (`off`): без привязки к названию, ≥30% пересечения опыта. См. §4 шаг 6, §7.

**«Где настраивается порог match?"**» → `search.llm_min_percent` в `config.yaml` (по умолчанию 30). В UI его нет. `match.min_percent` — внутренний для LLM-взвешивания, не порог отбора. См. §7.

**«Как добавить компанию в чёрный список?»** → UI: карточка «Фильтр компаний» → «Не откликаться» (по строке). Или `config.yaml` → `filters.blacklist_employers`. См. §6, §7.

**«Как откликаться только на определённые компании?»** → `filters.whitelist_employers` (в UI «Только эти компании»). Если непусто — отклик только на них. См. §7.

**«Как изменить модель LLM?»** → `config.yaml` → `llm.ollama.model` (Ollama) или `llm.openai.model` (внешний API). После смены Ollama-модели: `ollama pull <model>`. См. §10.

**«Как использовать OpenAI / внешний API?»** → `llm.provider: openai` (или `fallback_to_api: true`), `export JOBHUNT_API_KEY=...`, при желании `llm.openai.base_url`. Внимание: данные уйдут на внешний API. См. §10.

**«Ollama не отвечает»** → `ollama serve`, `ollama pull qwen2.5:7b`, проверьте `jobhunt doctor`. См. §12.

**«Бот пишет "не авторизован"»** → Войдите в окне `jobhunt ui` (вкладка hh.ru) или `jobhunt login`. Сессия в `~/.jobhunt/browser-profile`, не в Chrome. См. §12.

**«Капча останавливает прогон»** → Это задумано. Пройдите капчу вручную в вкладке hh.ru, перезапустите. Бот не обходит антибот. См. §12.

**«0 откликов»** → Проверьте `resume_title`, `resume_search_url`, `llm_min_percent`, режим фильтра, заполненность профиля. Запустите dry-run и смотрите «Пропущено». См. §12.

**«Где отчёт?»** → `reports/Otclicki_hh_User_*.xlsx` или карточка «Отчёты» в UI. Листы: Отклики / Требует действия / Пропущено. См. §11.

**«Какое имя у отчёта / можно ли переименовать?»** → `Otclicki_hh_User_<дата>_<время>.xlsx`. Имя задаёт код (`jobhunt/core/report.py`), пользователь не управляет.

**«Как остановить прогон?»** → UI: «Стоп» (мягкая остановка после текущей вакансии). CLI: Ctrl+C. См. §6.

**«Можно ли запускать два прогона одновременно?»** → Нет. Второй `/api/run` вернёт 409 «Уже выполняется запуск». См. §6.

**«Что в листе "Требует действия"?»** → Вакансии с тестом/опросом/капчей/нестандартной формой — отклик не отправлен, нужно вручную. См. §11.

**«Как заполнить профиль?»** → Скопируйте `context/resume-profile.example.md` → `context/resume-profile.md`, заполните роль/опыт/навыки/критерий. Или загрузите через UI. Контакты туда писать не нужно. См. §8.

**«Где подпись письма / мои контакты?»** → `config.yaml` → `profile.signature` и шаблон `context/cover-letter.md` (плейсхолдер `{signature}`). Не в профиле. См. §8, §9.

**«Как изменить стиль писем?»** → `skills/stop-slop-writing.md` (скопируйте из `.example.md`). Действует, когда письмо генерирует LLM (шаблон письма пуст). См. §9.

**«Что если у меня не PM/PO?»** → Поменяйте `resume_title`, `search.queries`, целевую роль и блок `blocker` в профиле. Логика «По роли» завязана на `resume_title` + `HARD_REJECT`-паттерны в `jobhunt/core/role_filter.py`. См. §7, §8.

**«Как добавить новую площадку (не hh.ru)?»** → Через `jobhunt/adapters/` (реализовать search/parse/apply), зарегистрировать в `search.site`. См. `AGENTS.md`.

**«Можно ли менять пороги/логику фильтров?»** → Только если пользователь явно попросил. Не меняйте молча — это влияет на то, куда уйдут реальные отклики. См. `AGENTS.md`.

**«Где найти технические детали / код?»** → CLI — `jobhunt/cli.py`, цикл прогона — `jobhunt/runner.py`, конфиг — `jobhunt/config.py`, фильтры — `jobhunt/core/filters.py`, роль-фильтр — `jobhunt/core/role_filter.py`, UI — `jobhunt/web/`, LLM — `jobhunt/llm/client.py`. См. `AGENTS.md`.

**«Как проверить окружение?»** → `jobhunt doctor`. Проверяет config, профиль, playwright, ollama, модель, профиль браузера. См. §5.
